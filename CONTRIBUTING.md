# Contributing
1) Open a Feature or Bug issue using the templates.  
2) Link your PR to the issue.  
3) Keep content in `_pages/`, data in `_data/`, layouts in `_layouts/`.  
4) CSS goes in `assets/css/style.scss` only.

This repo represents the **Memphis DC capital node of FlightLine**. Keep naming, gates, DSX references, and Proof Stack language consistent so other city grids (St. Louis MIC, Chicago MIC, etc.) can re-use this pattern.
